# LifeOS — Complete Product & Design Specification

### Personal University Archive · Second Brain · Memory Vault

---

## 1. Vision & Philosophy

LifeOS is a **private, single-user website** — not a social platform, not a SaaS product. It exists to preserve four years of university life, knowledge, and personal growth in one connected system.

It merges five things that are normally scattered across different apps:

1. A **university memory archive** (photos, videos, notes, moments)
2. A **personal diary / journal**
3. A **smart knowledge base** (second brain)
4. A **digital academic dashboard** (subjects, GPA, exams)
5. A **life timeline** that keeps growing after graduation

**Core principle:** nothing here is designed to be shared, liked, or scrolled by anyone but the owner. Every design and workflow decision should optimize for *personal meaning over time*, not engagement.

## 2. Design System

### 2.1 Visual Direction

| Attribute | Direction |
|---|---|
| Theme | Glassmorphism, dark-mode only |
| Mood | Minimal, premium, calm — not flashy |
| Depth | Slight 3D — tilt, parallax, layered blur, never cartoonish |
| Reference points | Apple Vision Pro spatial UI, Arc Browser's soft floating panels |
| Motion | Deliberate and smooth, never bouncy or "AI-generated default" |

### 2.2 Color Palette

**Backgrounds (the canvas the whole app sits on):**

| Name | Hex | Use |
|---|---|---|
| Void | `#07080F` | Page background, deepest layer |
| Navy Deep | `#0D1023` | Section backgrounds |
| Navy | `#12152C` | Card base tone (under glass blur) |

**Accent palette (the personality of the app):**

| Name | Hex | Use |
|---|---|---|
| Aurora Cyan | `#6CE5D8` | Primary accent — progress, links, highlights |
| Soft Violet | `#9B86F2` | Secondary accent — branding, gradients |
| Coral | `#F2879B` | Warnings, urgent tasks, emotional/mood markers |
| Gold | `#E8C27A` | Achievements, "in progress" states |

**Text:**

| Name | Hex | Use |
|---|---|---|
| Text Hi | `#F1F2FA` | Headings, primary text |
| Text Mid | `#A7ACC8` | Body copy |
| Text Low | `#5E6386` | Captions, timestamps, disabled states |

**Glass surfaces:**

* Panel fill: `rgba(22,26,48,0.55)` with `backdrop-filter: blur(22px) saturate(140%)`
* Border: `rgba(255,255,255,0.09)` (resting), `rgba(255,255,255,0.18)` (hover/active)
* Inner highlight: a 135° white gradient at ~7% opacity fading to transparent, to simulate light hitting glass from the top-left

**Rule:** Never use pure black (`#000`) or pure white (`#fff`) anywhere — always the tinted navy/text values above, so the whole UI feels like one coherent material instead of generic dark mode.

### 2.3 Typography

| Role | Typeface | Usage |
|---|---|---|
| Display / Headings | Space Grotesk | Page titles, hero numbers, nav brand |
| Body | Inter | Paragraphs, descriptions, UI labels |
| Data / Mono | JetBrains Mono | Dates, timestamps, stats, code snippets |

Letter spacing on headings is slightly tightened (`-0.02em`) for a more "engineered" premium feel rather than default web type.

### 2.4 Motion Language

| Moment | Behavior |
|---|---|
| Page/section switch | Fade + slight upward slide (~10px), 500ms, ease-out curve |
| Card hover | 3D tilt following cursor position (max ~6°) + subtle lift |
| Background | Slow-moving aurora gradient blobs + drifting particle dust, always ambient, never distracting |
| Scroll reveals | Fade-up for cards, parallax depth for hero imagery |
| Loading state | Glass cube or liquid blob animation, not a spinner |

**Rule of restraint:** one major animated "signature" moment per page (e.g., the constellation hero on Home, the glowing spine on Timeline). Everything else stays quiet so the signature moment actually reads as special.

## 4. Home Dashboard

**Purpose:** a single screen that answers "where am I right now, in life and in the semester."

### Contents & Logic

| Widget | Data shown | Logic |
|---|---|---|
| Greeting | Time-aware greeting + today's date | `if hour < 12 → "Good morning"`, etc. |
| Semester progress | % of semester elapsed | `(today − semester_start) / (semester_end − semester_start)` |
| Upcoming tasks | Next 3 assignment/exam deadlines | Pulled from Assignment + Exam tables, sorted by `due_date ASC`, filtered `status != completed` |
| Recent memories | Last 3 memories added | `ORDER BY created_at DESC LIMIT 3` |
| Recent notes | Last 3 notes added | Same pattern as above, from Notes table |
| Today's mood | One-tap mood picker | Writes a `mood_log` row keyed by today's date; overwrites if one already exists for today |
| Photo of the day | Random or pinned image from Memories | Default: random select from memories tagged with today's date in past years ("on this day"); fallback: most recent photo |
| On This Day | Memories from 1/2/3 years ago today | `WHERE MONTH(date)=MONTH(today) AND DAY(date)=DAY(today) AND YEAR(date) < YEAR(today)` |

## 5. Memories System

### 5.1 Data Model

Every memory entry has:

```
Title
Description
Date
Time
Location
Semester (auto-tagged from date range)
Tags (multiple)
Attachments (images / video / audio / text)
```

### 5.2 Timeline View

* Memories render as **zig-zag cards** alternating left/right along a vertical glowing spine — mirrors the visual metaphor of a "journey," not a flat feed.
* Sort order: chronological, newest or oldest first (user toggle).
* Each card shows: date, title, short description, tags, and a thumbnail if attachments exist.
* Clicking a card opens a full memory detail view (lightbox-style, glass modal).

### 5.3 Calendar View

* Standard month grid.
* **Heatmap logic:** each day's cell opacity/glow scales with `memory_count_for_day`. Light tint = few memories, deep glow = many. This turns the calendar into a visual "density map" of life.
* Clicking a day opens a side panel listing memories + notes + events logged that day.

### 5.4 Gallery View

* Display modes: Masonry grid (default), Horizontal carousel, Album view, Fullscreen viewer.
* Filter by: semester, tag, date range, media type.

### 5.5 Year / Semester / Event Views

* **Year View:** groups memories under Year 1–4.
* **Semester View:** sub-groups under Semester 1–8, matching the university journey structure below.
* **Event View:** memories tagged to a specific recurring or named event (e.g., "Hackathon 2024") collapse into one event card with all related memories nested inside.

### 5.6 University Journey Structure

```
Year 1 → Semester 1, Semester 2
Year 2 → Semester 3, Semester 4
Year 3 → Semester 5, Semester 6
Year 4 → Semester 7, Semester 8
```

This structure is the backbone that Memories, Notes, and University Dashboard data all key off of — every record can optionally inherit a `semester_id`.

## 6. Smart Notes System

### 6.1 What it stores

Plain text, website links, Instagram reels, YouTube videos, GitHub repos, movie recommendations, Linux tips, programming concepts, future plans — essentially anything worth remembering that isn't a "memory" in the emotional/event sense.

### 6.2 Groups (fixed top-level categories)

```
Programming · Movies · Linux · Web Design · University · Dreams · Funny · Ideas · Projects · Life
```

A note belongs to exactly **one group** but can carry **unlimited tags**.

### 6.3 Tags

* Free-form, unlimited, many-to-many with notes.
* Example tag set on one note: `website`, `3d`, `animation`, `gsap`, `threejs`.
* Tags double as a secondary navigation/filter layer across the whole Notes section.

### 6.4 Search Engine

* Searches across: title, description, tags, group, date.
* **Fuzzy search + typo correction + relevance ranking** — implemented via Fuse.js or FlexSearch, indexed client-side since this is a local-first app.
* Search is global — same search bar in the top bar can search Notes, Memories, and Achievements simultaneously, with results grouped by type.

## 7. University Dashboard

### 7.1 Modules

```
Subjects · Attendance · Assignments · Lab Reports · Exams · Results · Credits · Semester Progress · CGPA Tracker
```

### 7.2 Assignment Manager

**Fields:** Title, Subject, Deadline, Priority, Status, Attachments.

**Workflow / states:**

```
Pending → In Progress → Completed
```

* Pending: created but not started.
* In Progress: user manually flips status when they start working.
* Completed: locks the deadline countdown and moves the item out of "Upcoming" on Home.
* Overdue is a derived state, not stored: `if status != Completed AND today > deadline → render as Overdue (coral highlight)`.

### 7.3 Exam System

* Categories: Midterm, Quiz, Final.
* Each exam has a **live countdown timer** rendered as `Xd Yh` until the exam date, recalculated on page load.
* Past exams auto-archive into Results once a grade is entered.

### 7.4 GPA Tracker

**Logic:**

```
Semester GPA = Σ(course_grade_point × course_credit) / Σ(course_credit)   — for that semester
CGPA = Σ(all semester grade points × credits) / Σ(all credits)            — running total
```

* Progress chart shows GPA trend line across completed semesters.
* "Credits completed" vs "credits required to graduate" feeds the Semester Progress bar shown on Home.

## 8. Achievement System

* Stores: certificates, competition wins, project milestones, internship offers, awards.
* **Timeline-based**, similar visual language to the Memories timeline but filtered to only achievement-type entries.
* Each entry: icon/badge, title, date, optional linked memory or note (e.g., link the "won the hackathon" achievement to the memory entry from that day).

---

## 9. Project Archive

* Stores: source code links, screenshots, descriptions, GitHub links.
* Each project can optionally link to: the semester it was built in, related notes (tech learned), and related achievements (if it won something or shipped).

---

## 10. Bucket List

* Simple checklist of "things before graduation" (e.g., Build a SaaS, Visit Cox's Bazar, Participate in a hackathon, Learn React, Publish an application).
* Each item: text, optional target date, checked/unchecked state.
* Completed items can optionally auto-suggest creating a linked Memory or Achievement entry ("You just completed a bucket list item — log this moment?").

## 11. Statistics Page

A single page that aggregates counts across the whole system, purely for reflection:

```
Total memories · Photos · Videos · Notes · Projects · Achievements
Assignments · Friends · Days recorded · Years completed
```

All values are simple `COUNT()` queries against their respective tables; this page has no write logic, only read/aggregate.

## 12. Daily Journal & Mood Tracker

### 12.1 Daily Journal Fields

```
Today's mood
Best moment
What I learned
What I watched
Photo of the day
Random thought
```

One journal entry per calendar day, editable until midnight (soft rule, not enforced technically — just a UX convention).

### 12.2 Mood Tracker

* Options: 😊 Happy · 😐 Normal · 😴 Tired · 😔 Sad · 😎 Excited
* One mood value stored per day (`mood_log` keyed by date, see Home Dashboard logic above).
* Charts: weekly, monthly, yearly aggregation — essentially a frequency count of each mood emoji per time bucket, rendered as a simple bar or line chart.

## 13. On This Day

* Surfaces memories from the same calendar day in previous years (1/2/3 years ago).
* Query logic detailed in §4 (Home Dashboard table).
* Purpose: passive nostalgia — surfaces meaningful content without the user having to search for it.

---

## 14. Future Letters

* User writes a message to their future self.
* Each letter has a **lock date** — common presets: Graduation, 2030, 2035, 2040 (also supports custom date).
* Letter is hidden/encrypted from view until `today >= unlock_date`; UI shows a "locked" glass card with a countdown until then.

---

## 15. Friends Archive

**Fields:** Name, Department, Birthday, First Met (date), Memories Together (linked), Photos.

* Acts as a lightweight relationship CRM — purely for the user's own record, not shared.
* "Memories Together" is a relation: any Memory entry can tag one or more Friends, and this view simply reverses that relation to show all memories involving a given friend.

---

## 16. Project Ideas Vault

* A holding pen for raw ideas before they become real Projects: startup ideas, app ideas, website ideas, AI ideas.
* Deliberately lower-friction than the main Notes system — quick capture, minimal required fields (just a title + one-line description), so ideas don't get lost to friction.

## 17. Settings

```
Theme · Wallpaper · Accent color · Animation speed · Font · Backup settings
```

* Theme/accent/font changes apply instantly via CSS variable swaps (no reload).
* Animation speed is a global multiplier on all transition durations — useful for users sensitive to motion, and respects `prefers-reduced-motion` automatically regardless of this setting.

## 18. Data, Storage & Backup

| Layer | Choice | Why |
|---|---|---|
| Local database | IndexedDB via Dexie.js | Fully local-first, no server dependency, private by default |
| Backup export | JSON and ZIP | JSON for raw data portability, ZIP for bundling attachments + data together |
| Backup import | Restores from the same JSON/ZIP format | Lets the user move the whole archive to a new device |

**Backup workflow:**

```
Export → serialize all tables to JSON → bundle attachments into ZIP → download
Import → user selects file → validate schema version → merge or replace local DB → reload app
```
